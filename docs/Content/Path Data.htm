Path Data

Intro

#20a Consignor*

The consignor is the contractual party of the transport contract. Add name and address of a consignor. When consignor data is unknown, declare consignor in descriptive way referring to data in a waybill.
For example: Maersk, in accordance with CIM.

#20b RU executing the carriage*

In national transports usually one railway undertaking runs the exceptional transport. In international transports more railway undertakings are involved. Occasionally two railway undertakings run train on one railway network. Add country code in the front of the name of operating railway undertakings.
For example: (AT) Rail Cargo Austria, (DE) DB Cargo.

#21 Dispatch station*.

Is the station where cargo loaded on the wagon starts it's journey. Indicate station's name, station DIUM code and infrastructure manager's code or county code. When the first mile starts on the private siding add the name of the siding. For larger cities make sure to name exact station's operational name, since there may be a number of station in the city.
For example: Villach Sued CCT [81-01990-1](OeBB Infra)

#22 Destination station*

Is the station where cargo is off loaded an handed over to consignee. Indicate station's name, station DIUM code and infrastructure manager's code or country code. When the last mile starts is run on the private siding add the name of the siding. For cities where there are several stations at the destination, make sure to define an exact station's operational name.
For example: Duisburg Hafen WRS [80- 22665- 4](DB InfraGO).

#24 Carriage arrangement*

Declare if forwarding takes place in a block train or in a wagon group.
Add maximum speed with which train can run.
For example: normal freight train with vmax. 100 km/h , special freight train with vmax. 80 km.h.

 

#27a Consignee*

Consigniee is the entity to which goods are sent to be sold on commission (the buyer). Add name and address of a consignee. When consignee data is unknown, declare consignee in a descriptive way referring to data in a waybill.
For example: Maersk, in accordance with CIM.

 

#27bFreight payer

A freight payer is an entity which signed a contract with the railway underacting for the transport of a train or a wagon group. Add name of the freight payer and add the contract number. When freight payer data is unknown, declare freight payer in a descriptive way referring to data in a waybill or leave the box empty.
For example: Company AB & BA, contract no. ABC-11111-2423.

#26 Transhipment port - harbor loading quay

For larger ports make sure to direct the train to the right group of unloading tracks. Name the harbor loading quay where the train will be loaded or off-loaded.
For example: Gdansk Port Polnocny , Rudowy (ore) quay.

 

#30Direction of travel - label O

Direction of travel matters in two situations. For road vehicles carried on wagons it is important to arrive at the station in the right direction of travel, so they can drive off the wagon. In such case direction of travel is described here, and label O is not used. For goods which stick out more to one side of the wagon than the other, label O is used to mark the points on the cargo which sticks out.
For example: Take over at the border in Arnoldstein (AT)/ Tarvisio (IT) facing forward.

 

#31Previous forwarding numbers

If you make prolongation of the permit with carriage conditions for exactly the same type of cargo and wagon set and on the same path but for the next yearly schedule, then name the previously used approval numbers for the transport. This information can be helpufll for the Infrastructure managers.
For example: DB Bza _____, CZ MZ______,

 

#32Loading date, delivery date

Loading date declare either as exact date, or refer to the calendar week, or a month when the transport is planned to begin. This information helps the infrastructure manager to better plan their work and allocate their resources. As practice shows, the delivery date is of less
For example: from 12.08.2024, from the 17/18 week 2024.
