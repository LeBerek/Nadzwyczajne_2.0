Wagon Data

The majority of railway undertakings run trains composed of wagons from their own wagon park. In such case the railway undertaking holds the role of the owner, the keeper and the entity in charge of maintenance of the wagons. Wagon check of the obligatory (with asterisk) and optional (if required) parameters is run briefly since all parameters and wagon interoperability status are known.

Other wagons in a train are either from other railway undertakings or from the private vehicle owners. In such case wagon parameters check requires more effort since not all parameters are at hand. For transporting a good on an wagon, the wagon parameters sources include:

  • wagon data sheet or data from a wagon catalog,
  • loading scheme (for open flat wagons),
  • wagon sketch (if missing in wagon data sheet),
  • wagon photo with all markings of the wagon (alternative).

Wagons and Vehicles are to be declared as a type of good when they are subject to commercial contract, or are newly constructed and are in the tests phase. Some vehicles are exempted from the requirement of interoperability and in international transports they run on an exceptional transport basis (for example historical vehicles).

#1aWagon / vehicle type

Declare wagon construction type, series, serial numbers, specify whether empty or loaded and etc. as applicable. For vehicles state construction type, number of units (for vehicles with multiple units), respectively manufacturer's number or vehicle's EVN. Advise EVN status, whether reserved or already assigned. For wagon groups, or multiple vehicles declare just the first six digits of the EVN number in the point 1a and consider adding a more detailed list in the point 28 (in case of a few vehicles) or as an appendix with a separate list to the application , when several wagons are to be declared.
For example: newly build Tank wagon Zacns / GATX type 3500, EVN 37 84 **** ***-* (reserved).

The points below describe basic and crucial wagons and vehicles parameters which are required for exceptional consignment permit application. These points are declared with simplified examples.

#2aType of wagon*

Describe class of wagon and type number as specified in the standard marking.
For example: Laaps 1260, Zacns 292, Sggrs 80’.

 #2bInternational usability

Declare for international transportation, and for national transport of the non-interoperable wagons.
For example: RIV or TEN GE.

 #3 Wheel-base / bogie pivot pin pitch* [mm]

Indicate the distance (a) between end axles of vehicles not fitted with bogies or between bogie centers.
For example: (a) 8600, (a) 10100, (a) 2x 10375.

 #4 Bogie wheel base* [mm]

Add the distance (p) between the outermost axles for wagons on bogies.
For example: (p) -, (p) 1800.

 #5 Number of axles*

Declaration depends on the construction of a wagon. Describe in detail the axle layout of the wagon
For example: 2, 4 (2x 2), 6 (3x 2), 8 (2x (2x 2)).

 #6aLength over buffers* [mm]

For articulate wagons (short coupled), the additional annotation is accepted.
For example: 27000 (2x 13000), 15140, 26390 (2x 13195).

 #6bFloor height above Top of Rail* [mm]

Parameter applies only to open wagons transporting goods which while placed on the wagon may foul the loading gauge.
For example: 1260, -, 1150.

 #7 Tare weight of wagon* [t]

Tare weight value from the wagon data sheet indicates the maximum of the tare weight for the particular wagon class and type number. Tare weight value from a photo indicates the actual wagon tare weight.
For example: 26.6, 21.5 t ± 2%, +/-26.0.

#7+8Total weight* [t]

The sum of a good's mass plus it's securing components and the wagon tare. Declare it with actual sum values or state the maximum weight in relation to the requested line load class. Observe relationship between the total weight and the wagon speed.
For example: max. 79,6; 90,0; 120 or max. allowed for lines class C3 (with vmax. 120 km/h).

#9 Load per meter* [t/m]

Declare the value indicated by wagon manufacturer or calculate the actual value of load per meter by dividing the total of the wagon’s dead weight plus the cargo weight, by the wagon’s length in meters. Alternatively state the maximum value of load per meter associated withe the line class reference value. For wagons with load grid it is sufficient to declare "normal", if allowed total weight in Point 8, for the line class is not exceeded.
For example: 3,0; 7,2 ( max. C-class), normal D3.

 #10Largest axle load* [t]

Enter a value indicated by wagon manufacturer or a calculated the actual value of load per meter. Alternatively you may declare the maximum value of load per meter associated withe line class reference values. For wagons with load grid it is sufficient to declare "normal", if allowed total weight for the line class is not exceeded.
For example: 20.0, normal D3.

Below example references vehicles which sizes conform with the preferred route. For wagons and vehicles which do not conform with preferred route gauge, explore the chapter on Out-of-Gauge Loads.

#19Comments on critical points

Declare the vehicle gauge profile with which wagon conforms. Where applicable declare the upper and lower reference profile conformity. Determine compliance with the kinematic or static gauge profile by reference to UIC leaflet 505-1 or EN 15273-2 as appropriate.
For Example: Wagon conforms with G1/GI1 reference profile, The wagon contour does not exceed the reference profile of the G2 kinematic gauge according to the EN standard.

Application declarations for the points 28 and 29 greatly vary in included details because they depend on the type of railway vehicle being transported. Therefore, these two points contain summary descriptions which are sufficient to declare parameters of a wagons with the goods. For more specific prompts for filling an exceptional transport application consult XXX in Handy Information Tab.

#28Miscellaneous -remarks on railway wagons and vehicles

In miscellaneous remarks declare breaks status (active / inactive), coupling and buffer status (when non-standard), wheel diameter dimension and type of suspension if these differ from the GCU and TSI regulations, minimum allowed curve radius of the wagon and other technical details of the vehicle, and operating conditions which may affect carriage conditions.
For example: Vehicle accompanied by manufacturer's employee, brakes switched on, smallest radius of curvature Rmin 150 m, wheel diameter ø 850 mm.

 #29Forwarding conditions

The forwarding conditions remarks refer to payload characteristic which affect how the consignment or a vehicle must be prepared for the transportation and how to examine the path so the transport may take place. For example, remarks may include information on payload earthing, on personnel with route knowledge or the knowledge of vehicle operation, on consignment / vehicle position in a train (behind locomotive, or at the end of the train). Other restrictions may include ban on wagon group separation or ban on propelling, hump shunting and passing over humps.
For example: include wagon at the end of the train, wagon group separation forbidden.

Point 33, is recalled for information purposes only, as it is occasionally declared in exceptional consignment applications.

#33Inspection requirement - trust based agreement

In the application declare if the transport is to be a hand over or a took over in trains which are subject to trust-based agreement (ATTI). Include information where the train inspection is required and by which entity.
For example: yes -train is a subject to ATTI agreement, inspection conducted at station Paskov by PKP Cargo International, no - at station Petrovice an inspection is required by CD Cargo.
